12. Best Practices e Raccomandazioni

12.1 Configurazione Sistemi

12.1.1 Nuova Integrazione

  1. Utilizzare template application di riferimento o TEMPLATE_RELEASE_1_0_0
  2. Configurare sempre RPA Primario prima della migrazione
  3. Verificare direttive provisioning dopo import
  4. Testare task aggregazione prima attivazione NCD
  5. Validare schema profili con stakeholder

12.1.2 Migrazione da IAM Engine

  1. Assegnare RPA Primario su IAM Engine prima migrazione
  2. Verificare file CSV generati da DMT
  3. Controllare alberatura cartelle e file account/profili
  4. Analizzare file "incongruences.csv"
  5. Eseguire Application Builder solo dopo verifica DMT
  6. Validare creazione task aggregazione
  7. Verificare attributi estesi schemi profili
  8. Confermare catalogo MIMIP
  9. Dismettere sistema su IAM Engine solo dopo test completi

12.2 Governance

12.2.1 Certificazioni

  • Utilizzare sempre tag per identificare campagne
  • Non annullare prima certificazione in Staging (perdita template)
  • Configurare notifiche reminder/escalation
  • Non modificare Exclusion Rule senza "Save As"
  • Sign-off obbligatorio per completare certificazione
  • Verificare auto-approvazione item non lavorati

12.2.2 Native Change Detection

  • Abilitare solo dopo migrazione completa
  • Configurare attributi da riconoscere coerentemente
  • Schedulare task aggregazione in orari non operativi
  • Monitorare Pending AS per errori ricorrenti
  • Documentare decisioni RPA Primario in commenti
  • Gestire profili non censiti tempestivamente

12.3 Provisioning

12.3.1 Gestione Errori

  • Analizzare sempre dettaglio transazione prima azione
  • Documentare motivazioni in campo commento
  • Preferire "Reinoltra Transaction" a "Forza Esito OK"
  • Monitorare disallineamenti post-forzatura
  • Verificare consistenza dopo ogni Pending AS risolta

12.3.2 Performance

  • Schedulare task aggregazione fuori orario operativo
  • Limitare richieste batch concurrent per applicazione
  • Monitorare code OSB per saturazione
  • Verificare timeout configurati adeguatamente
  • Pianificare manutenzioni in finestre concordate
